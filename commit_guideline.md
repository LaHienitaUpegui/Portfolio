# Commit Guidelines

## Language

All commit messages must be written in **English**.

## Format

Every commit must include:

1. **Title**: Short, clear summary of the change (max 50 characters)
2. **Body**: Brief description of what was done and why (if needed)

## Best Practices

- Use imperative mood (e.g., "Add feature" not "Added feature")
- Start with a verb: Add, Fix, Update, Refactor, Remove, etc.
- Be specific about what changed
- Reference issues/tickets when applicable

## Examples

```
Add user authentication flow

Implemented login and logout functionality using JWT tokens.
```

```
Fix navigation link on mobile devices

Updated CSS to properly handle responsive navbar.
```

```
Update project card images

Added Naliia webapp and website screenshots.
```
